Guía rápida

Claude Code.

Todo lo que necesitas saber, en una página. Sin relleno, sin tutoriales de quince pasos, sin humo.

01

Empezar

Instalas una vez, autenticas una vez, luego te olvidas de que existe.

# Instalar
npm install -g @anthropic-ai/claude-code

# Lanzar desde la raíz del proyecto
claude

# Primera vez
/login
Regla #1. Lanza claude siempre desde la raíz del proyecto. El directorio actual es su mundo.
02

Dar contexto

Claude sin contexto adivina. Claude con contexto ejecuta. La diferencia es un archivo: CLAUDE.md.

Generarlo automáticamente

/init   # lee tu proyecto y genera CLAUDE.md

Qué va dentro

  • Comandos clave — cómo correr tests, build, lint
  • Convenciones — estilo, nombres, estructura
  • Decisiones no obvias — por qué X en vez de Y
  • Zonas peligrosas — archivos que no debe tocar
Jerarquía. ~/.claude/CLAUDE.md global → ./CLAUDE.md proyecto → ./sub/CLAUDE.md submódulo. Todos se cargan.
03

Cómo pedir

No es magia, es comunicación. Peticiones concretas devuelven resultados concretos.

Vago

"arregla el login"

Concreto

"el login falla con emails en mayúsculas — normaliza a lowercase en auth.ts antes de comparar"

Patrones que funcionan

  • Adjunta archivos con @ — @src/auth.ts @tests/auth.test.ts
  • Inyecta contexto con ! — !git diff HEAD~3 antes de preguntar
  • Di qué NO hacer — "no toques la migración, solo el modelo"
  • Define "hecho" — "cuando los tests de auth pasen"
04

Los modos

Alterna con Shift Tab. El modo activo aparece en la esquina inferior.

ModoComportamientoCuándo
Default Pide permiso antes de cambios Día a día
Accept Edits Aplica ediciones sin preguntar Tareas largas de confianza
Plan Solo investiga y planifica, no toca Antes de cambios grandes
Bypass Sin permisos Sandbox o contenedor
Plan mode primero. Para tareas de más de 30 minutos: entra en Plan, pide el plan, revísalo, sal y ejecuta. Ahorras horas de deshacer.

Flujo diario

# 1 · Abrir proyecto
$ cd mi-proyecto && claude

# 2 · Contexto rápido
!git status
!git log --oneline -10

# 3 · Pedir con archivos concretos
@src/api/users.ts @src/api/users.test.ts
añade paginación cursor-based, mantén compat con ?page=

# 4 · Revisar antes de merge
/review

# 5 · Compactar si la sesión es larga
/compact

Interrumpir sin romper

  • Esc — parar lo actual, mantener el plan
  • Esc Esc — editar tu último mensaje
  • Ctrl C — cortar todo (último recurso)

07

Subagentes

Claude puede delegar en otras instancias de sí mismo. Cada una con su contexto, corriendo en paralelo.

De fábrica

ExploreExplora el código sin ensuciar tu sesión
PlanDiseña estrategias antes de implementar
general-purposeTareas multi-paso con contexto propio

Los tuyos

/agents   # crear, editar, borrar

Cada agente es un Markdown en .claude/agents/ con frontmatter que define nombre, descripción, herramientas y modelo.

08

Hooks

Ejecuta comandos shell cuando pasan cosas: linter, notificaciones, backups, lo que quieras.

// .claude/settings.json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Edit|Write",
      "hooks": [{
        "type": "command",
        "command": "prettier --write $CLAUDE_FILE_PATHS"
      }]
    }]
  }
}

Eventos

  • PreToolUse — antes de cada herramienta (puede bloquear)
  • PostToolUse — después de cada herramienta
  • UserPromptSubmit — cuando mandas mensaje
  • Stop — cuando Claude acaba de responder
  • SessionStart — al arrancar
/hooks   # UI para configurar sin tocar JSON
09

MCP Servers

Extiende Claude con herramientas externas: GitHub, Postgres, Slack, Linear, tu propia API... Cualquier cosa con un servidor MCP.

# Añadir un servidor
claude mcp add github -- npx -y @modelcontextprotocol/server-github

# Ver los tuyos
/mcp
Idea. Los MCP son herramientas, no magia. Claude los usa igual que Read o Bash. Añade uno cuando te descubras copiando datos entre sistemas a mano.

Regla de oro. Si repites una instrucción dos veces, ponla en CLAUDE.md. Si repites una tarea dos veces, hazla skill o hook. Claude mejora cuando tú dejas de repetirte.
